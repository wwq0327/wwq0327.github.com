OpenClaw 安家记：从 Windows 笔记本搬到 Mac mini 上的第一天

Mac mini 到了，养龙虾的日子正式开始。

先把电脑接上。没想到，这第一步就卡住了。

Mac mini 没有 USB 接口。第一次进系统，最好的连接方案是苹果官方的键盘和鼠标。我没有。键盘倒是有一把，Type C 接口，刚好能插。但没有鼠标，根本没法操作。后来突然想起来，我有一套切换设备。把 Mac mini 用 Type C 线连到那个设备上，设备上可以接无线键盘和鼠标。问题才解决，这才登录进系统，进了桌面。

十二点多了，先吃饭。吃完开始装 OpenClaw。第一次装，还要装苹果自己的命令行工具，装 Node。那些装完，才能装 OpenClaw。设置倒简单，因为以前搞过，APK、飞书机器人，都是现成的，复制粘贴就行。

接下来是把原来跑在 Windows 上的整个 Workspace 目录复制过来。二十多兆，主要是配置文件、脚本，还有插件。压缩后十多兆，用 OneDrive 同步，下载、解压、复制过来。迁移完成。

当然还有一些细化工作。以前在 Windows 上，工作目录路径不同，有的要手动调，有的直接让它自己检查、自己调整。大概花了一个多小时，系统基本调好了。记忆系统、工作目录，指定好，它知道去哪儿查文件，就 OK 了。

下午在外面晒太阳，用起来很正常，没什么问题。主要就是和它聊聊天，设置、优化一些记忆内容。

晚上回到家，想检查定时任务。喊它查有没有问题，它说没问题。去控制台一看，定时任务一条记录都没有。很奇怪。喊它设两条，它说设了，一查，还是没有。后来出现了很奇怪的现象：问它定时任务怎么设置，它就直接掉了，等一会儿也不重启。

没办法，把 Claude Code 调出来，让它帮忙查。它怀疑是 Node 版本问题——V24，建议装 V22。按建议卸载 V24，装了 V22。装起来还是比较容易的，因为前面配置过，Workspace 里的配置都在，重新配 API 和飞书的 key 就行。结果设定时任务的时候，还是莫名其妙断线。喊 Claude Code 查，它（Claude Code）也查不出问题。

后来仔细查发现：每次从飞书发消息过去，就会断。在控制台的聊天窗口发，不管是查定时任务还是设定时任务，都没问题。怀疑是飞书的问题。

跑到飞书官网去查。果然，3月28日发的那个版本有 bug，发消息会把 OpenClaw 卡掉。卸载了飞书插件，删掉配置，重新安装。好，可以用，定时任务没问题了。

装好之后，把 Mac mini 关机，搬到书架上，位置移远一点。再开机，接上键盘显示器，登录进去，然后就退出。现在就是一台主机加一根电源线。只要不停电，就可以一直跑。开通了 SSH，平时在游戏机上操作完，就可以直接 SSH 登录过去处理事情。

OpenClaw 用起来并不是那么省心。后面还涉及升级、工具的搭建，还有很多事要做。感觉真的像养小龙虾，觉得不费事，实际上比养个小宠物都还费事。
